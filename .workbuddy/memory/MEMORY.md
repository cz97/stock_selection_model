# 长期记忆

## 用户投资框架（核心身份）

**投资理念**：在超景气行业的拐点前夜布局，跑在市场前面，赚企业基本面与逻辑发生重估的钱。

**关键词解读**：
- **"拐点前夜"**：渗透率从 0→1 即将进入 1→10 的阶段；或景气度刚见底回升
- **"跑在市场前面"**：在市场还在质疑、分歧、低覆盖时建仓
- **"赚逻辑重估的钱"**：不是只赚 EPS 增长的钱，而是赚估值重估（PE 15x → 40x）的钱
- **"事件"**：指行业/个股的现象级事件（如国家大力发展商业航天），不是财报公告、业绩预告、解禁这类常规公司事件

**三击验证框架**：政策 + 技术 + 需求，三者齐全才算超景气赛道。

**覆盖市场**：A股 + 港股 + 美股。

**体系流派补充**：
- 属于**价值投机**，不是价值投资（吃息派）
- 选股路径分为"现象级事件提前埋伏"和"困境反转"两类；后者适用于政策强制干预（限产/环评/矿权）或纯市场厮杀（行业出清）的传统行业
- 反例约束：白酒/房地产不适用困境反转（没有需求回暖）

## Expert 使用定位（已锁定）

**真实用法**：低频 + 深度 + 一次性的"投研对话伙伴"，不是量化工具，不是自动扫描。
典型场景：
- 用户有闲钱时，扔主题/股票进来让 AI 走一遍框架
- 用户准备月度/季度投资计划时，提前筛一批股票关注
- 频率：每周/每月级，不是每天

**明确不做**：
- ❌ 定时自动扫描
- ❌ 全自动量化策略 / 回测
- ❌ 实时监控推送
- ❌ 脚本化批量处理

## 数据源真相（2026-06 更新）

### ✅ 核心数据栈

- **WorkBuddy 内置股票数据（westock-data）**：已验证可用，作为主数据源。
  - `quote`：行情、估值快照、市值、PE/PB/PS、涨跌幅
  - `finance`：A股/港股/美股财报，多期利润表/资产负债表/现金流
  - `notice` / `ncontent`：公告列表和公告 PDF 链接
  - `risk`：A股风险事件
  - `consensus`：A股一致预期
  - `profile`：公司简况
  - `shareholder`：股东结构
  - `sector`：板块/概念成份股
- **web_fetch**：抓年报/公告正文、经营计划、未来展望、港交所披露易、SEC EDGAR。
- **Web Search**：产业信息、市占率、反向证伪；只采事实，不直接采主观结论。

### ❌ 不作为主路径

- **旧 Python 数据方案**：已被内置股票数据替代；保留为历史验证记录，不再写入 Skill 主流程。
- **finance-data-retrieval / Tushare 209 接口**：外部鉴权不稳定，不作为候选。
- **Tushare.pro 付费 API**：付费，当前不需要。
- **雪球 / 知乎专栏**：反爬或观点污染严重，不作为事实源。
- **neodata-financial-search**：自然语言查询可用但结构化历史数据不稳，不作为主力栈。

## 七维信息结构（2026-06 更新）

| 维度 | 数据源 | 处理方式 |
|---|---|---|
| 1. 行业景气度 | Web Search + 用户输入 | 只判是否过门槛；政策/技术/需求三击验证 |
| 2. 业务纯度 | 内置股票数据 + 年报 | 目标景气赛道占比过低则 pass |
| 3. 动态预测估值 | 内置股票数据 + 年报指引 + 一致预期 | 亏损公司用经营计划/出货/产能推预计 PS；仅参考，不机械扣分 |
| 4. 龙头地位 | 年报 + Web Search | 看市占率、技术代差、代际卡位，不看营收/市值排名 |
| 5. 市场辨识度 | Web Search + 板块成份 | 人气 + 名字 + 历史标签 + 稀缺性，权重不高 |
| 6. 风险值 | `risk` / `notice` / 反向证伪 | 一票否决：减持/诉讼/监管/治理/地缘制裁等 |
| 7. 流动性环境 | 指数行情 + Web Search | 市场背景，不直接决定个股标签 |

**真实结构**：不是七维平铺打分，而是"三个前置门槛 + 信息整理 + 四档标签"。
- 前置门槛：行业景气度、业务纯度、风险值
- 关键信号：**双拐点共振** = 行业拐点 + 企业基本面拐点同时出现
- 输出：🟢重点关注 / 🟡可打野 / ⚪观察等待 / 🔴不予配置

## 项目术语（公开版）

- **双拐点共振**：行业拐点 + 企业基本面拐点同时出现。替代旧称，作为当前项目公开术语。
- **行业单拐点**：只有行业拐点，企业基本面尚未确认。
- **剑宗 vs 气宗**：一波流（半年到一年止盈，小仓位） vs 长期格局（可容忍大幅回调，主要仓位）
- **自主可控 ↔ 海外龙头镜像**：中芯↔台积电、寒武纪↔英伟达、速腾↔AEVA、北方稀土↔MP稀土
- **超配策略**：拐点出现时对个股两倍做多，只出手一次不添油；逻辑兑现后止盈超配部分
- **龙一龙二策略**：龙头先配、等差距拉开后买龙二当彩票
- **困境反转**：独立的选股路径，与现象级事件并列

## 公开化/匿名化约束

用户希望把本项目作为个人项目发布：
- 文档和 Skill 中不再出现外部个人称呼或外部归属痕迹
- 旧称全部替换为公开版术语："双拐点共振"、"行业单拐点"
- `raw/` 原始研究材料、`.workbuddy/memory/`、测试脚本和历史调试结果不应公开发布
- `result/` 用于沉淀每次个股、行业或事件分析报告，属于可复用投研资源，但不作为公开发布内容，已加入 `.gitignore`

## 产物沉淀规则

每次完成 `stock-screening` 分析后，必须把完整 Markdown 报告写入项目根目录 `result/`：
- 个股分析：`result/{个股名}_个股分析.md`
- 行业分析：`result/{主题名}_行业分析.md`
- 事件分析：`result/{事件名}_事件分析.md`
同名文件默认覆盖为最新结论；如用户要求保留历史版本，再追加日期后缀。
