# 长期记忆

## 用户投资框架（核心身份）

**投资理念**：在超景气行业的拐点前夜布局，跑在市场前面，赚企业基本面与逻辑发生重估的钱。

**关键词解读**：
- **"拐点前夜"**：渗透率从 0→1 即将进入 1→10 的阶段；或景气度刚见底回升
- **"跑在市场前面"**：在市场还在质疑、分歧、低覆盖时建仓
- **"赚逻辑重估的钱"**：不是赚 EPS 增长的钱，是赚估值重估（PE 15x → 40x）的钱
- **"事件"**：指的是**行业/个股的现象级事件**（如"国家大力发展商业航天"），**不是**财报公告、业绩预告、解禁这类公司事件

**三击验证框架**：政策 + 技术 + 需求，三者齐全才算超景气赛道。

**覆盖市场**：A股 + 港股。

**体系流派补充**（2026-04 补）：
- 属于**价值投机**，不是价值投资（吃息派）
- 与"困境反转"并列：作者把选股路径分为"现象级事件提前埋伏"和"困境反转"两类；后者适用于政策强制干预（限产/环评/矿权）或纯市场厮杀（行业出清）的传统行业，典型案例 2019 年煤炭、2025 年锂矿/光伏
- 反例约束：白酒/房地产不适用困境反转（没有需求回暖）

## Expert 使用定位（已锁定）

**用户的真实用法**：低频 + 深度 + 一次性的"投研对话伙伴"，不是量化工具，不是自动扫描。
典型场景：
- 用户有闲钱时，扔主题/股票进来让 AI 走一遍框架
- 用户准备几月投资计划时，提前筛一批股票关注
- 频率：每周/每月级，不是每天

**因此明确不做**：
- ❌ 定时自动扫描（/scan-events 不做）
- ❌ 全自动量化策略 / 回测
- ❌ 实时监控推送
- ❌ 脚本化批量处理

**因此不需要**：akshare-one-mcp / tushare.pro / 定时任务 / 本地数据库。

## 数据源真相（2026-04-21 更新）

### ✅ 核心数据栈（已验证稳定）

- **akshare（A 股）**：稳定，首选
- **akshare（港股）**：**可用但接口签名不统一**，必须用正确参数：
  - `stock_financial_hk_report_em(stock=代码, symbol="利润表", indicator="年度")` — 参数名是 `stock`
  - `stock_financial_hk_analysis_indicator_em(symbol=代码, indicator="年度")` — 参数名是 `symbol`
  - `stock_hk_hist(symbol=5位代码)` — 不带 .HK 后缀
  - 偶发反爬，`time.sleep(5)` 重试多数能过
- **Web Search + web_fetch（同花顺 F10 / 东方财富）**：机构聚合官方数据的网页，不反爬
- **港交所披露易 hkexnews.hk**：权威原文，年报/业绩公告首选

### ❌ 禁用清单

- **finance-data-retrieval**（Tushare 209 接口）：2026-04 排查确认外部通道不通；即使在 AI 会话内可调，也不稳定，多次证明不可用。**全面禁用，不作为候选**。
- **Tushare.pro 付费 API**：付费，用户明确不走
- **雪球 xueqiu.com**：严格反爬，`web_fetch` 拿到的是验证页；个人观点污染严重
- **知乎专栏 zhuanlan.zhihu.com**：自媒体污染，观点倾向强
- **neodata-financial-search**：之前记为"可用"，但在实际 /score 流程里没有验证到对港股历史数据的可靠性，不作为主力栈

### 🚨 处理反爬的正确姿势

**不要一遇反爬就换数据源**（那是假问题）：
1. `RemoteDisconnected` → `time.sleep(5)` 重试 2-3 次
2. 失败继续 → 换 akshare 的另一个接口（同数据东财/新浪双源）
3. 还不行 → `web_fetch` 抓同花顺 F10 页面（不反爬）
4. 兜底 → Web Search 取主流财媒引用的官方公告

**工作模式**：Expert = 一份 SKILL.md + 打分模板 + 参考资料。AI 每次会话按模板走，在会话里调工具。不做脚本化。

## 七维打分体系（2026-04 更新版）

| 维度 | 数据源 | 自动化方式 |
|---|---|---|
| 1. 行业景气度 | Web Search + 人工判断 | AI 综合三击信号分档（政策 / 技术 / 需求） |
| 2. 业务纯度 | finance-data: `fina_mainbz` | 会话内 AI 执行 |
| 3. 动态预测估值（原"历史估值位置"） | akshare 财报 + 年报指引 | **亏损公司必须从年报"经营计划"提取未来营收预期算预计 PE，仅作参考不机械扣分** |
| 4. 龙头地位 | finance-data + Web Search | **看市占率+技术代差+代际卡位，不看营收/市值排名** |
| 5. 市场辨识度 | finance-data: `concept` + Web 研报覆盖 | 人气 + 名字 + 历史标签 + 稀缺性四要素（原话，非"预期差"），权重不高 |
| 6. 风险值 | finance-data: `anns_d` | 一票否决：定增/减持/诉讼/财务造假 |
| 7. 流动性系数 | neodata 实时 | 相对指数点位的换手强度（2/3/3.5/4万亿分档，v1 用作者原话口径） |

**打分表的真实结构（SKILL.md 实现时必须遵守）**：
- 七维不是平铺加权，是"门槛 + 打分"两阶段
- **Step 1 前置门槛（任一不过 → 直接 pass，不进入打分）**：
  - 行业景气度 ≠ 0（作者反面案例：中国中免景气度 0 直接不能买）
  - 业务纯度 不能过低（伪概念股排除）
  - 风险值 无一票否决项
- **Step 2 加权打分**：剩下四维（估值位置 / 龙头 / 辨识度 / 流动性）相乘/相加 × 流动性系数
- 这样能避免"某股七维算 65 分看着能买、实际不能碰"的假阳性

**重要纠错**：
- 原总结文档"没有止盈的说法"是字幕翻译错误，应是"**没有止损的说法**"。作者实际持仓积极止盈（逻辑兑现/破坏时），只是不设机械止损。
- 原流动性阈值"0.8/1.5万亿"已过时（2024 年），现在 2万亿是冰点、3万亿健康、3.5-4万亿危险。
- 行业景气度排第一不是因为权重最高，是因为它是"A股 5000 家的漏斗入口"——人类和 AI 都不可能一家家看，只能先过景气行业再研究个股。

## 独创概念速查

- **笨韭双击**（笨笨的韭菜的简称，作者自称）：行业拐点 + 企业基本面拐点同时出现。从头到尾忽略 PE/PB，在企业大额亏损时买入。对应"笨韭单击"（只有行业拐点）。
- **剑宗 vs 气宗**：一波流（半年到一年止盈，~10% 仓位） vs 长期格局（可容忍腰斩回调，主要仓位）
- **自主可控 ↔ 遥遥领先镜像**：中芯↔台积电、寒武纪↔英伟达、速腾↔AEVA、北方稀土↔MP稀土
- **超配策略**：拐点出现时对个股两倍做多，只出手一次不添油；逻辑兑现后止盈超配部分
- **龙一龙二策略**（2026 补）：龙头先配、等差距拉开后买龙二当彩票
- **困境反转**（2026 补）：独立的第四类选股路径，与现象级事件并列

## 关键原话口径备查

- "100-300 亿市值是科技标的最佳区间；<30 亿必避开；<100 亿慎重"
- "稀缺性的权重大于毛利"（A股定价规则）
- "剔除净现金后估值 <10 倍"（价值底线）或 "(FCF+回购+派息)/市值 > 2× 无风险利率"
- "景气投资不设机械止损，但允许止盈"
- "大仓位长期持有；打野（30-40 万）可止损止盈"

## MCP 结论（最新，覆盖旧结论）

**不装外部 MCP 的真实原因**：用户是低频深度使用，不需要脚本化数据管道。
既然不脚本化，finance-data 会话内调用就够用，MCP 的"外部可调用"优势用不上。
如果未来升级为定时扫描 / 脚本化分析，再考虑装 akshare-one-mcp（免费且外部可调）。
